# CLAUDE.md — remove_notebooklm_watermark

Offline, single-file HTML app that removes the bottom-right "NotebookLM" watermark
from images / PDF / PPTX. Reimplements the CV approach of
`Albonire/notebooklm-watermark-remover` (Python + OpenCV) in vanilla JS/Canvas.

## The deliverable is generated, not hand-edited

`dist/notebooklm-watermark-remover.html` is **built** — do not edit it directly.
Edit the sources, then rebuild:

```bash
python3 build.py        # src/ + vendor/  ->  dist/notebooklm-watermark-remover.html
```

`build.py` inlines `src/css/styles.css`, the UMD vendor libs, and the app modules
into `src/index.template.html` (placeholders `<!--INJECT:CSS|VENDOR|PDF_WORKER|APP-->`),
and embeds `pdf.worker.min.js` as base64 (turned into a Blob URL at runtime by `app.js`).

## Hard constraints (offline `file://`)

- **No ES modules, no `fetch`, no servers.** Everything is IIFE + a global `NLM`
  namespace (same pattern as the `SPC/` project). Third-party libs must be **legacy
  UMD** builds (pdf.js 3.x legacy, pdf-lib, JSZip) — ESM builds break on `file://`.
- pdf.js worker is provided as a Blob URL (see `app.js setupPdfWorker`); never point
  it at an http(s) URL.

## Source layout (`src/js`, load order matters)

`core.js` (namespace, bus, config, canvas/image/download helpers) → `engine.js`
(gray, box blur, morphology, connected components, **fillHoles**, neighbour-inpaint,
patch-heal) → `detect.js` (`buildMask`: dark-candidate + components + cluster, optional
text template) → `process.js` (orchestration: `cleanFullImage`, `cleanRoiScaled`,
`cleanWithUserMask`, `maskFromSpec`) → handlers `image.js` / `pptx.js` / `pdf.js` →
`manual.js` (modal mask editor → normalized shapes) → `ui.js` → `app.js`.

Each handler exposes `representative(file)` (image for the manual editor) and
`process(file, cfg, manualSpec, onProgress)` → a result `{found,name,blob,previewBefore,previewAfter,...}`.

## Hard-won gotchas (already fixed — don't regress)

- **neighbour-inpaint must expand past the mask bbox** (`cfg.inpaintExpand`), or a
  *solid* mask (e.g. a manual rectangle) has no interior anchors and changes nothing.
- **fillHoles** is required so solid icons / letter interiors get covered (the
  local-background subtraction only catches outlines of solid marks).
- **`roiRightBias`/`roiBottomBias` kept ~0.28** (not the reference's 0.45/0.35): the
  tighter bias clipped the icon left of the text on the no-template path.
- **PDF**: render a *generous* corner for clean reconstruction, but overlay only the
  tight watermark bbox via `pdf-lib drawImage` (preserve the rest of the doc). Map
  device px ↔ user coords with `viewport.convertToViewportPoint` and the mediabox origin.
- Default reconstruction is **neighbour-inpaint** (seamless on gradients); `patch-heal`
  is opt-in (textured backgrounds) and can leave a tile seam on gradients.

## Verify

```bash
node test_fixtures/verify.js     # headless Chrome, expects "RESULT PASS"
```

Drives the built file with synthetic fixtures (PNG/PDF/PPTX, auto + manual), checks
bottom-right darkness before/after, and asserts zero console errors + pdf.js worker is a
`blob:` URL. Fixtures are generated by the snippets in that dir (PIL / pdf-lib / python-pptx).
`node_modules/` + `package.json` here are dev-only (puppeteer-core for verification).
